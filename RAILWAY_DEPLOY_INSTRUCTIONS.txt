# Railway Deployment Instructions

## Prerequisites
1. Install Git: https://git-scm.com/download/win
2. Install Node.js and npm: https://nodejs.org/
3. Install Railway CLI: `npm install -g @railway/cli`
4. Create Railway account: https://railway.app/

## Step-by-Step Deployment

### 1. Login to Railway
```bash
railway login
```

### 2. Initialize Git Repository (if not already done)
```bash
git init
git add .
git commit -m "Fixed phone_utils import and Railway configuration"
```

### 3. Initialize Railway Project
```bash
railway init
```

### 4. Set Environment Variables
```bash
railway variables set DJANGO_SETTINGS_MODULE="freelance_platform.settings.production"
railway variables set SECRET_KEY="django-insecure-change-this-in-production"
railway variables set DEBUG="False"
railway variables set ALLOWED_HOSTS="*.railway.app"
```

### 5. Deploy to Railway
```bash
railway up
```

## Post-Deployment Steps
1. Check Railway dashboard for deployment status
2. Note your application URL (usually https://your-app-name.railway.app)
3. Set up database migrations (handled automatically by Procfile)
4. Configure any additional environment variables as needed
5. Test the application functionality

## Troubleshooting
- If deployment fails, check Railway logs
- Ensure all configuration files are committed to git
- Verify environment variables are set correctly
- Check that the WSGI application path is correct
